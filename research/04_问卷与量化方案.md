## 3. 问卷结构（模块）

### M1 筛选与分群（必答，用于分组）
- 你的角色？A 从业者 / B 代理商（管理≥2人）/ C 中小企业主
- 团队规模、行业、近 3 月是否投过广告、是否用过 AI 生图工具

### M2 行为
- 出图频率、常用工具（多选）、主要场景（社媒/电商/门店/提案）

### M3 痛点普适性（5 点李克特：非常不同意→非常同意）
题项直接来自 H1–H6：
- M3-1 我点"生成方案"后，**最期待立刻看到广告原图**（H1）
- M3-2 如果只出文字方案不出图，我会觉得"没做完"（H1）
- M3-3 **没看到图就被冻结/扣除积分**，会让我不安（H2）
- M3-4 我更愿意"先看到图、满意再付费"（H2）
- M3-5 多岗位自动流水线对我**很有用**（H3，预期 B 高 C 低）
- M3-6 我希望自己控制**材质/风格**，而非全自动（H5）
- M3-7 我希望"原图/环境效果图/矢量"能**明确分开切换**查看（H4）

### M4 结果形态偏好（排序/多选，H4）
- 请排序：广告原图 / 真实环境效果图 / 矢量·图层导出 / 可直接发布海报，你最常用哪类？

### M5 工作流价值（分群交互，H3）
- 对 B：团队人效提升预期？对 C：是否觉得"太复杂"？

### M6 积分 / 付费意愿（H2 / G3）
- 你能接受"生图前冻结、成功再扣"吗？（接受/不接受/视金额）
- 偏好付费方式：按次积分 / 包月 / 按交付件数
- 价格锚：一次完整出图交付，你愿付 ___ 元 / ___ 积分（开放区间）

### M7 SUS（标准 10 题，0–100）
- 使用标准 SUS 量表（奇数正向、偶数反向）。

### M8 NPS + 开放
- 多大可能推荐给同行（0–10）；"最想让我们改的一件事？"

## 4. 分析方法（详见 `05`）

| 分析 | 方法 | 用途 |
|---|---|---|
| 描述统计 | 频率、均值、中位数 | 总体痛点强度 |
| 分群差异 | **卡方**（类别×分群）、**ANOVA / Kruskal-Wallis**（李克特均值跨群） | 验证 H3/H4/H5 是否分群显著 |
| 基线 | **SUS** 与基准 68 比较 | G1 可用性达标判断 |
| 驱动因素 | **回归**（满意度 ~ 痛点项） | 找出最影响满意度的杠杆 |
| 定价 | 价格锚分布 + 分群交叉 | G3 定价区间建议 |

> 显著性水平 α=0.05；效应量报告（Cramér's V / η²）避免"显著但无意义"。

## 5. 质量与伦理

- 匿名采集，不收集姓名/手机号等直接标识（除非抽奖需联系，单独脱敏存储）。
- 题文中立，不预置"AdCraft 很好"等引导。
- 投放前小样本（n=20）试填，修正歧义题。
- 数据加密存储，按留存期销毁；研究报告仅呈现聚合结果。

- 问卷在 Phase 2 与可用性测试**同期或稍后**投放，使"定性主题 → 定量普适性"形成三角验证。
