# 分析框架与结果解读（AdCraft AI 用户研究）

> **配套**：`00_研究计划总览.md`（总纲）、`01`–`04`（证据/访谈/可用性/问卷）
> **作用**：规定"拿到数据后怎么分析、怎么得出结论、怎么避免误读"。本文件同时服务 Phase 1（定性）与 Phase 2（定量），并贯穿 Phase 4（持续）。
> **核心原则**：三角验证 + 假设公示 + 证据分级 + 反确认偏误。

---

## 1. 总体分析哲学

| 原则 | 含义 |
|---|---|
| **三角验证** | 定性主题 ↔ 定量普适性 ↔ 行为数据，三者收敛才判高置信 |
| **假设公示** | H1–H6 前置公开，分析时主动寻找"推翻证据"，防确认偏误 |
| **证据分级** | 强（≥2 独立源收敛）/ 中（单源但高度一致）/ 弱（单源待验证） |
| **负向优先** | 反例、弃用、差评必须纳入，不得只报利好 |

---

## 2. 定性分析：主题分析六步（Braun & Clarke）

| 步 | 动作 | 在本项目落点 |
|---|---|---|
| 1 熟悉数据 | 通读转写 + 观察笔记 | 每场 24h 内完成 |
| 2 初始编码 | 开放编码，贴标签 | 用下方编码本 |
| 3 寻找主题 | 聚类相关编码 | 每周亲和图 |
| 4 审查主题 | 主题是否贴合数据 | 抽 2 场复核 |
| 5 命名主题 | 抽象+具象命名 | 如"确定性预期缺口" |
| 6 撰写 | 证据→洞察→建议 | 研报格式 |

**编码本（初始版，随数据扩展）**
```
H1_确定性预期    用户期待"输入→端到端出图"的确定性
H2_付费焦虑      未看到产出即冻结积分的不安
H3_工作流价值    多角色流水线对不同分群的价值/负担
H4_结果形态心智  原图/环境图/矢量 的心智模型差异
H5_可控性        材质/风格结构化选择器的需求
H6_假成功风险    mock/假成功对信任的摧毁(红线)
痛点_找不到结果  分散面板导致迷路
情绪_不信任      旅程中信任下滑节点
```

**质量保障**
- **双编码**：2 名研究员独立编码，一致性 **Cohen's κ ≥ 0.7** 方采纳。
- **饱和度**：新访谈不再涌现新主题 → 达饱和（本设计 6 人/群即接近）。
- **成员校验**（member checking）：初步结论回和用户抽检确认。

## 3. 定量分析：描述 + 推断

| 分析 | 方法 | 判读 |
|---|---|---|
| 描述 | 频率、均值、中位数 | 痛点强度排序 |
| **SUS** | 标准公式（奇数正向/偶数反向，×2.5） | **≥68 可接受，≥80 优秀**；分群对比 |
| 分群×类别 | **卡方检验**（观察 vs 期望频数） | H3/H4/H5 是否分群显著（α=0.05） |
| 李克特跨群 | **ANOVA**（方差齐）/ **Kruskal-Wallis**（非参） + 事后检验 | 群间态度差异 |
| 驱动因素 | **回归**（满意度 ~ 痛点项） | 标准化 β 排序，找最大杠杆 |
| 效应量 | Cramér's V / η² | 防"显著但无意义" |

> 所有推断报告**效应量 + 置信区间**，不只报 p 值。

## 4. 三角验证矩阵（核心交付逻辑）

| 假设 | 访谈证据 | 问卷普适性 | 行为数据 | 置信度 | 结论动作 |
|---|---|---|---|---|---|
| H1 确定性预期 | T1 原话… | M3-1/2 均值… | 生成→出图漏斗… | 强/中/弱 | 采纳/再验 |
| H2 付费焦虑 | T4 表达… | M3-3/4… | 冻结→放弃率… | | |
| H3 工作流价值 | 分群差异… | M3-5×分群卡方 | 工作流采用率 | | |
| H4 结果形态 | T3 迷路… | M4 排序… | T3 找结果时长 | | |
| H5 可控性 | 访谈… | M3-6… | 选择器使用率 | | |
| H6 假成功 | （禁止触发） | — | 上线后信任指标 | 红线 | 工程必须修 |

**判读规则**：三源收敛 → 强置信，可直接进路线图；单源 → 弱，需补数据；冲突 → 优先查方法偏差。

## 5. 旅程情绪曲线

- 输入：可用性情绪节点（🟢🟡🟠🔴）+ 访谈情绪叙事 + 行为放弃点。
- 输出：**当前态情绪曲线**，标注断裂点（对照 `01` §2）。
- 用法：与"目标态旅程"对比，断裂点 = 最高优先级修复项（多为 H1/H2/H6 相关）。

---

## 6. 画像证据索引规则

- 每个画像特质须有 **≥3 条证据**（quote / 数据点 / 行为），并标注来源 ID。
- **升级规则**：假设级（来自 `01`）→ 经验验证级（Phase 1/2 每特质获 ≥3 证据）→ 置信画像（三角验证强）。
- 禁止"平均用户"陷阱：始终以分群呈现，差异即洞察。

## 7. 优先级矩阵（影响 × 成本）

```
          高实施成本
             │  重要且难 → P1(排期) 
  高影响    │
             │  重要且易 → P0(立即做) ★
────────────┼───────────── 低实施成本
             │  次要且难 → P2(评估)
  低影响    │
             │  次要且易 → P3(顺手做)
          低实施成本
```

- **影响** = 痛点强度 × 受影响用户量 × 与营收/留存关联度（来自三角验证）。
- **成本** = 研发/设计/内容估算。
- 读图：落在"高影响×低成本"格 = 立即做；"高影响×高成本" = 排期并拆 MVP。

## 8. 研报结构与沟通

**研报大纲**
1. 执行摘要（3 条最关键洞察 + 建议）
2. 研究方法与置信度说明
3. 用户画像（证据索引）
4. 当前态旅程与断裂点
5. 假设验证表（H1–H6 逐条）
6. 分群差异
7. 优先级矩阵与路线图建议
8. 局限与后续研究

**沟通句式**：坚持"**证据 → 洞察 → 建议**"，每条建议可追溯到数据。
**对齐对象**：产品（路线图）、设计（交互）、研发（断链修复）、增长（定位/定价）。

---

## 9. 结果解读红线（防误读）

1. ❌ 不把 mock/占位数据当作真实用户行为。
2. ❌ 不对小样本（问卷 <30/群）做过度推断。
3. ❌ 不只对利好下结论，负向案例必须呈现。
4. ❌ 不把"相关性"说成"因果"（回归仅示关联）。
5. ✅ 任何跨群结论必须报告效应量与样本量。
6. ✅ H6（假成功）是上线红线，属工程项，研究侧仅监控，不"验证"其可接受性。

## 10. 与持续阶段（Phase 4）的衔接

- 上线后行为漏斗（生成→生图→精修→导出→复购）、NPS、应用内微反馈，持续回填本框架的"行为数据"列。
- 季度用户日记补充定性纵深，形成"研究—上线—再研究"闭环。
