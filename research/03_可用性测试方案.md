# 可用性测试方案（AdCraft AI 用户研究 · Phase 2 验证）

> **配套**：`00_研究计划总览.md`、`01_桌面研究与假设.md`（H1/H2/H4/H5）、`05_分析框架与结果解读.md`
> **目标读者**：主持人、观察员、产品/设计/研发负责人

---

## 1. 测试前提（关键红线）

| 前提 | 说明 | 责任方 |
|---|---|---|
| **P0 断链必须先修复** | 前端串联 `/image-jobs` 真实生图、编辑器接真实资产与积分、brief 结构对齐（见 MVP 审计 P0） | 研发 |
| **测试对象 = 可走通的闭环** | 测的是"修复后的真实闭环"，不是 mock | 研发+研究 |
| **禁用 mock 当真实** | 绝不以硬编码假图/假成功展示给被测用户（防 H6 污染） | 研究 |
| **生图协议需修通** | 默认 async 致网关 404/403，须改 sync 或兼容（见生图逻辑对比 §5） | 研发 |

> ⚠️ 若 P0 未修复，测试应转为"概念可用性 + 期望测量"，并在报告中明确标注"闭环未通，数据仅反映预期"。

---

## 2. 测试目标

1. 测量"一句话 → 出图 → 精修 → 合成环境 → 导出 →（积分）"闭环的**任务完成度**。
2. 验证假设：**H1**（端到端确定性预期）、**H4**（三类结果心智模型）、**H5**（材质/风格可控性）、**H2**（积分冻结感知）。
3. 建立 **SUS 可用性基线**（目标 ≥68）。
4. 暴露"用户在哪一步失去耐心/不信任"（呼应 `01` 旅程情绪曲线）。

---

## 3. 参与者

| 项 | 方案 |
|---|---|
| 数量 | **8–12 人**（Nielsen：5–8 人发现 ~80% 问题；取上界覆盖分群） |
| 分群 | A 从业者 3–4、B 代理商 3–4、C 中小企业主 3–4 |
| 无障碍 | **≥2 名残障用户**（视/肢/认知），验证 WCAG 2.1 AA |
| 招募 | 同 `00` §4.3；激励等价 |

---

## 4. 任务场景（基于真实闭环）

| 任务 | 场景描述 | 成功标准 | 验证假设 | 度量重点 |
|---|---|---|---|---|
| **T1 出图** | "你是奶茶店老板，想做一张促销海报。输入需求，生成方案并**看到 AI 生成的广告原图**。" | 成功提交生图并看到结果图 | H1 / F1 | 完成率、等到图的时长、放弃点 |
| **T2 精修** | "把背景换成门店照片，加一行促销文字，调整位置。" | 完成背景替换+文字+位移 | H5 | 错误数、求助次数 |
| **T3 找结果形态** | "客户要一张'放在真实门店里'的效果图 / 或能继续编辑的矢量图层，请找到并导出。" | 找到对应结果并导出 | H4 / F6 | 是否迷路、找结果时长 |
| **T4 走通闭环+积分** | "完整走一次出图并查看你的积分变化。" | 理解冻结→扣减，无困惑 | H2 / F9 | 积分感知、焦虑表达 |

---

## 5. 度量指标

**定量**
- 任务完成率（二分类 + 协助完成/部分完成分级）
- 任务时长（每任务秒数）
- 错误数 / 求助次数 / 放弃率
- **SEQ**（任务后单题易用性 1–7）
- **SUS**（10 题，0–100，基准 68）

**定性**
- 出声思考（think-aloud）原话
- 情绪节点（🟢🟡🟠🔴）与"不信任/放弃"触发时刻
- 访谈后 3 条最强 quote

**系统**（若可用）
- 行为日志：生成→生图→精修→导出漏斗、积分变动时点

---

## 6. 测试脚本（60 min）

| 时段 | 内容 |
|---|---|
| 0–5' | 欢迎、知情同意、说明 think-aloud、轻松背景聊 |
| 5–15' | 基线：当前出图方式、用过的 AI 工具、期望 |
| 15–50' | T1–T4 任务（每任务后 SEQ + 简短追问） |
| 50–55' | SUS 问卷 + 整体印象 |
| 55–60' | 收尾：最意外的点、一句话建议 |

**追问技巧**：卡住时不给答案，问"你预期下一步发生什么？""现在你想怎么做？"；探测 H1 确定性预期与 H2 积分焦虑。

---

## 7. 数据收集与处理

- 录屏+录音（经许可）；转写标注。
- 定量录入表格：参与者ID / 分群 / 每任务 完成度·时长·错误·SEQ / SUS。
- 定性进入亲和图与编码本（见 `05`）。
- 无障碍用户单独记录障碍类型与严重度。

---

## 8. 判断标准（如何读结果）

| 指标 | 达标 | 预警 |
|---|---|---|
| 闭环完成率 | ≥85% | <60% → 断链/引导严重问题 |
| 等到图时长(T1) | 用户可接受（定性无弃） | 出现"以为没反应"→ 进度反馈缺陷 |
| T3 找结果 | 无迷路 | 多数迷路 → H4 成立，需 Tab 化 |
| SUS | ≥68 | <68 → 整体重设计 |
| 积分焦虑(H2) | 少负向 | 多"未看到图就被扣"不安 → 调整冻结时机 |

> 结果将汇入 `05` 三角验证，与问卷普适性、行为数据交叉，收敛为高置信结论。
