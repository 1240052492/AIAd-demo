# AdCraft AI（神笔）MVP 实现审计

> 审计日期：2026-07-10
> 对照基准：用户合并「本地分析 + 原型图 + 方案文档 + Claude 评审」后的 MVP 开发方案
> 审计范围：`apps/server`（后端）、`apps/web`（前端）实际代码

## 结论速览

**后端骨架扎实、正确**：认证、积分（冻结/扣减/退回事务）、`/api/ai/brief`、`/api/ai/prompt`、`/api/image-jobs`、环境图上传、项目/画布、Admin 后台、Worker 链路全部存在且可跑通（已用 curl 端到端验证）。

**但收费闭环在「前端」是断的**：
1. 前端**从不调用** `/api/ai/prompt` 与 `/api/image-jobs` —— 用户点「生成方案」后只生成 brief + 建项目，**广告原图从未真正生成**。
2. 编辑器 `useEditor.ts` 的素材/任务状态/冻结积分**全部硬编码为 mock 数据**（注释原话「后端暂未提供素材列表接口」），不读取后端真实资产与任务。
3. Claude 返回的 `imagePrompt` 被丢弃，未用于生图。

**一句话**：后端把"生成原图 / 查任务 / 扣积分"的能力都做好了，但前端既没把生图任务提交出去，也没把结果接回来——MVP 收费闭环目前走不通。

## 二、核心断链（必须修才能收费）

### 断链 1：前端从不提交生图任务
- 证据：`grep "/image-jobs"` 在 `pages/ hooks/ components/` 无任何命中；`api.ts` 虽定义了 `imageJobs.create`，**但没人调用**。
- 后果：`Home` 生成方案只做 `brief + 建项目 → 跳编辑器`，**广告原图永远不生成**。
- 后端 `/image-jobs` 本身 OK（curl 实测入队→Worker 消费→积分退回全通过）。

### 断链 2：编辑器全量 mock，不接后端
- 证据：`apps/web/src/hooks/useEditor.ts`
  - L122-139 `envImages` / `aiImages` = **硬编码 SVG dataURL**（注释「后端暂未提供素材列表接口」）
  - L141 `jobStatus = 'succeeded'`（硬编码）
  - L119 `frozenCredits = 30`（硬编码）
- 后果：即便生图成功，编辑器也显示假图、假状态、假积分；`GET /projects/:id/assets`（`projectService.getAssets` 已存在）从未被前端调用。

### 断链 3：imagePrompt 被丢弃
- 证据：`Home` 调用 `aiApi.brief` 后只取 `briefRes.data.brief` 存入 `briefJson`，**未取也未用 `imagePrompt`**；且 `image-jobs` 本应接收 prompt，但前端根本没调它。

> 这三个断链叠加 = **"客户一句话 → 生图 → 出效果图 → 扣积分"的收费闭环当前跑不通**。

## 三、接口偏差明细

### A. `/api/ai/brief` 返回结构不符方案
方案要求 Claude 输出 JSON：
```json
{ "brief": {}, "missingQuestions": [], "imagePrompt": "", "productionNotes": [], "riskWarnings": [] }
```
实际 `anthropic.service.ts` `generateBrief` 返回：
```json
{ "brief": { "...", "missingQuestions": [...] }, "imagePrompt": "" }
```
偏差：
- `missingQuestions` 被塞进 `brief` 内部（应为**顶层**）
- **缺顶层 `productionNotes`（施工说明草稿）与 `riskWarnings`（风险预警）** —— 而方案明确要 Claude 做"施工说明草稿"
- `imagePrompt` 有返回（好），但前端未消费

修复方向：改 `AE_SYSTEM_PROMPT` + `normalizeBrief`，输出 `{brief, missingQuestions(顶层), imagePrompt, productionNotes, riskWarnings}`。

### B. `/api/ai/prompt` 返回形态
实际返回 `{ prompts: string[] }`（1-4 个风格变体）。方案只说"优化生图 prompt"，多变体属合理增强，但**前端未串联**，应接入"生图前优化"步骤或移除该接口避免误导。

### C. `AssetService` 无独立模块
方案列 `AssetService：上传图、生成图、导出图`。实际拆散在：`projectService.uploadAsset` / `image.worker.ts`（写生成资产）/ `composition.service.ts`（写合成资产）/ `projectService.exportProject`。**功能覆盖，但无统一模块**，不符合方案命名与职责划分。

### D. 后端 `/projects/:id/export` 占位死代码
`projectService.exportProject` 写白底 1024 占位 PNG / 极简 SVG / 最小 PDF，前端**不走它**（前端用 Fabric 客户端导出）。建议删除或标注，避免误用。

## 四、超出 MVP（"先不做"却已实现）

方案「先不做」清单 vs 实际：

| 先不做项 | 实际状态 |
|---------|---------|
| 多智能体岗位工作流 | ⚠️ **已实现**：`workflow.engine.ts` + `skills/`（6 个 Agency-Workflow 角色）+ `POST /api/ai/workflows/run` + Admin `workflows` CRUD。且 `/workflows/run` 冻结并消耗 4 积分 |
| 自动精准透视融合 | 未做（MVP 用手动套图，符合） |
| 自动高质量矢量化 | 未做（SVG 导出是 Fabric 原生矢量，符合） |
| 自动报价/订单/团队/多租户 | 未做（符合） |
| 复杂材质物理渲染 | 未做（符合） |
| PS 插件/无限画布/ComfyUI | 未做（符合） |

**额外超出**：Admin 还实现了 `provider-configs`、`settings`、`workflows` CRUD —— 多于方案"最小版"要求（user/credits/tasks/provider/templates）。属于合理富余，但工作流相关应评估是否隐藏（见下）。

> 风险：工作流引擎属于"先不做"，却占位了 `skills/` 与独立入口，且 `/workflows/run` 会消耗积分。在 MVP 阶段建议**隐藏入口**，避免误导用户消耗积分走一条未列入首期的链路。

## 五、修复优先级建议

### P0（阻断收费闭环，必须修）
1. **前端串联生图**：`Home` 生成方案后 → 调 `/ai/prompt`（或直用 brief.imagePrompt）→ 调 `POST /api/image-jobs` → 轮询 `GET /api/image-jobs/:id` → 把结果写入项目资产。
2. **编辑器接真实数据**：`useEditor` 删除 mock，改为 `GET /projects/:id/assets` 拉取 `aiImages`/`envImages`，`jobStatus`/`frozenCredits` 从真实任务/积分接口取。
3. **`/api/ai/brief` 输出对齐方案**：补全 `productionNotes`、`riskWarnings`，`missingQuestions` 提到顶层。

### P1（MVP 体验与合规）
4. `Home` 增加**材质/风格结构化选择器**（或确认用自由文本即可，需在原型中明确）。
5. 把"三类结果"做成**明确 Tab/视图切换**（广告原图 / 真实环境效果图 / 矢量·图层导出），而非分散面板。
6. 把 `imagePrompt` 一路传到生图任务（断链 3 修复）。

### P2（清理与决策）
7. 删除后端占位 `/projects/:id/export` 死代码，或在前端统一走客户端导出。
8. 评估 `composition` Worker：是否暴露独立 `POST /api/compose` 接口（让"一键合成真实环境"可用），还是保持 MVP 手动套图。
9. 评估工作流（`/api/ai/workflows/run` + `skills/` + Admin workflows）：MVP 阶段**隐藏入口**，待闭环跑通后再启用。
